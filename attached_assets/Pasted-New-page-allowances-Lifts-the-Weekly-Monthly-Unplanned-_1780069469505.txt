PAGE LAYOUT:

1. Header: "Allowances" / "Weekly, monthly, and unplanned spend — reviewed."

2. Time-range selector at top: "Week of May 24–30, 2026" / "Month of May 2026" + prev/next chevrons to look back. Two tabs above the chevrons: WEEK / MONTH. Default to WEEK / current week on first load.

3. Below the selector: THREE bucket summary cards in a row, one each for Weekly, Monthly, Unplanned. Each card shows:
   - Bucket name (e.g. "Weekly allowance")
   - Big number: actual spend in this period
   - Sub-line: planned allowance for this period
   - Variance: actual − planned, color-coded (under = green, over = red)
   - Mini progress bar: actual vs planned

4. Below the cards: an expandable list of transactions belonging to those buckets for the period, grouped by bucket and by category (for the Weekly bucket which has sub-buckets — Groceries / Dining / Entertainment / Misc per the existing TransactionWeeklyBucket enum). Click a category row to expand and see the actual transactions that rolled into it (description, date, amount).

   Default state: all bucket groups collapsed. Click to expand. Match the Variance Summary popover treatment from /debrief — same visual language.

5. Below the transaction breakdown: a small "Over/under summary" block. Plain text. "This week you came in $42 under your $300 weekly allowance" etc. No AI yet — just deterministic text.

LOOK-BACK: when user clicks back-chevron, page loads previous week (or previous month if in MONTH tab). Locked weeks read from the existing weeklyDebriefs table render the locked snapshot's totals; non-locked weeks compute live.
