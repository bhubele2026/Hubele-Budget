Feature: make the Category Variance numbers clickable. Clicking a Planned or Actual amount opens a popover showing the individual line items that make up that number. This is the whole point of the variance table — I see "Groceries actual $848.95" and I want to click it and see the 4 transactions behind it.

PART 1 — Backend: include line-item breakdown in each category bucket.

In weeklyDebrief.ts, the byCategory buckets currently carry only { categoryId, plannedAmount, actualAmount, varianceAmount }. Extend each bucket with two arrays:

  plannedItems: Array<{
    recurringItemId: string | null;
    name: string;          // e.g. "Weekly Groceries"
    amount: number;        // the forecast amount for this occurrence
    forecastDate: string;  // YYYY-MM-DD
  }>;
  actualTxns: Array<{
    txnId: string;
    description: string;
    amount: number;        // signed
    date: string;          // pending date (occurredAt fallback occurredOn), YYYY-MM-DD
    matchedToPlan: boolean; // did this txn match a planned item, or is it unplanned?
  }>;

Populate them in the same aggregation loop that already computes plannedAmount/actualAmount — you're already iterating the plan occurrences and transactions per category, just collect them into these arrays instead of only summing. The sum of plannedItems[].amount must equal plannedAmount; the sum of actualTxns[].amount (abs) must equal actualAmount. Add a cheap assertion or comment noting that invariant.

Update the OpenAPI spec for the bucket shape + run orval codegen.
