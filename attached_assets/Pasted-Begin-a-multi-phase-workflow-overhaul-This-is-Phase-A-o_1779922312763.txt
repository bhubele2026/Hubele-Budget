Begin a multi-phase workflow overhaul. This is Phase A of 6 (A, B, C, D, E, F). Each phase is independent — land this one cleanly before we move to B.

Goal of Phase A: remove auto-matching of Chase transactions to planned forecast items. Going forward, NOTHING matches automatically. Every match is an explicit user action. This unblocks the rest of the overhaul (manual Send-to-Review gate, lingering past-due plans, weekly Debrief page, etc).

CHANGE 1 — Identify and disable the auto-match path

Search the api-server codebase for where incoming Chase transactions get matched to planned recurring items without user intervention. Likely locations:
  - artifacts/api-server/src/lib/plaidSync.ts (during sync, possibly auto-resolves forecast items)
  - artifacts/api-server/src/lib/forecastAutoMatch.ts or similar
  - artifacts/api-server/src/routes/forecast.ts (POST endpoints that bulk-create resolutions)
  - Anywhere forecast_resolutions rows are INSERTed with status='matched' WITHOUT an explicit user action triggering it

For each auto-match path you find:
  - Add a feature flag check: const AUTO_MATCH_ENABLED = false; (hardcoded for now, can become an env var or settings toggle later)
  - When the flag is false, skip the auto-match block entirely
  - Log a one-line note at info level when the skip happens, so we can verify it's running

Important: DO NOT remove the auto-match CODE — wrap it in the flag. We may want to bring it back later as an explicit user-toggled "AI auto-match" feature in Phase D. Just gate it so it doesn't fire today.
