FIX — THREE PARTS:

PART 1 — Server-side snapshot pointer repair in startupChaseDeadItemsCleanup.ts.

After all the orphan-pointer cleanup is done, run one more repair: for TARGET_HOUSEHOLD, if forecast_settings.bankSnapshotAccountId IS NULL AND there is exactly one Chase plaid_account (institution_name ILIKE 'chase'), repoint:
  UPDATE forecast_settings
    SET bank_snapshot_account_id = <that chase plaid_accounts.id>,
        bank_snapshot_name = <that account.name>,
        bank_snapshot_mask = <that account.mask>,
        bank_snapshot_source = 'plaid',
        updated_at = NOW()
    WHERE user_id = <owner>
      AND bank_snapshot_account_id IS NULL;

Leave bank_snapshot_balance and bank_snapshot_at alone — those will be filled on the next refresh-from-plaid or balance sync. The pointer is enough to make the page default to Chase.

Idempotent: re-running converges to no-op once the pointer is set.

PART 2 — Auto-pick Chase as the default account on the Chase page.

In artifacts/h2budget/src/pages/transactions.tsx line 319, change defaultAccountKey logic:
  // If a Chase Plaid checking account exists, default to it.
  // Falling back to "manual" only when there are zero Plaid accounts.
  const chaseAccount = (forecastData?.plaidCheckingAccounts ?? []).find(
    (a) => (a.institutionName ?? "").toLowerCase().includes("chase")
  );
  const defaultAccountKey =
    bankSnapshot?.accountId
    ?? chaseAccount?.id
    ?? "manual";

This makes the page land on the Chase account even when the snapshot pointer is missing — the visible signal the user wants.

PART 3 — Remove the View account dropdown when there is exactly one Plaid checking account.

In artifacts/h2budget/src/pages/transactions.tsx around line 2244-2254, the dropdown visibility gate is `totalOptions > 1`. Tighten it: also exclude "Manual entries" from the option count unless the user is currently viewing manual rows.

Specifically: drop the `shouldShowManualPickerOption` evaluation from the totalOptions count when it's only adding Manual as a secondary option. The dropdown should be hidden whenever there's a single real Plaid account.

Don't change anything if there are 2+ Plaid checking accounts (multi-checking households still need the picker). Just clean up the single-account case.

Also: drop the "Manual entries" SelectItem from the dropdown entirely. If someone has manual rows mixed with a Plaid account, the dropdown still shows the Plaid account; manual rows render in the fallback path. We do not need a separate "manual" view.
