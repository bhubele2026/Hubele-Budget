   These are heuristic. Don't claim 100% accuracy. If the category-name of the txn ALSO matches (e.g. "Dining & Coffee" or "Groceries" or "Gas, Maintenance & Parking"), prefer that as a stronger signal. Specifically:
   - dining: match if merchant OR category name contains "dining"
   - groceries: match if category.name === "Groceries"
   - gasStation: match if category.name contains "gas" or "fuel"
   - coffee: match if merchant matches AND amount < $15 (excludes a bulk Starbucks pantry order from polluting the coffee tile)
   - amazon: merchant match only
   - onlineShopping: merchant match only

3. STREAK COMPUTATION:
   - noDining: walk daily from trackingStart through today. For each day, was there a dining txn? Streak = consecutive days WITHOUT a dining txn. Current = streak ending today. Longest = max streak observed across the full range.
   - coffeeFree: same logic for coffee-keyword + amount<$15 filter
   - Both streaks measured in days, not weeks
   - Edge case: if the user has zero dining txns in the entire range, currentDays = daysCovered, longestDays = daysCovered, longestEndDate = today
