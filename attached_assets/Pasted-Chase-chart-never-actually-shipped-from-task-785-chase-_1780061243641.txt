DESIGN — Chase page chart (artifacts/h2budget/src/pages/transactions.tsx).

WINDOW
- Start = max(2026-05-01, current month start).
- End = today + 12 months.

THREE SERIES, ALL WEEKLY (Sun–Sat buckets)

1. historicalActual — weekly end-of-week Chase checking balance from window start through the last completed week before today. Uses the existing snapshot + per-account dedupe path (makeChaseBalanceAtEndOf / computeBalanceAtEndOf / scopeChaseTransactions in artifacts/h2budget/src/lib/chaseEndingBalance.ts and chaseScope.ts). If a balanceAtEndOfDate(isoDate) helper doesn't exist as a sibling of the month-bucketing one, generalize the existing helper — same anchor + net-change logic, just summed by date instead of by month.

2. forecastFromToday — starts at today's actual Chase balance. Projects forward weekly Sun–Sat using the same projection /forecast consumes (server cashSignal via useGetForecast — forecastData has the daily series). Aggregate those daily projection points into weekly buckets through the end of the window.

3. actualFromToday — seeded with today's actual balance as the only point on day one. As future days pass, new Plaid balance points populate forward weeks naturally.

ALL THREE share today's actual balance as their anchor: visually the historical line meets today's balance, then actualFromToday continues from that same point AND forecastFromToday diverges from it.

RENDERING

- Reuse BalanceTrendChart (artifacts/h2budget/src/components/account-page/balance-trend-chart.tsx). If the Amex single-series mode shipped, extend to multi-series; if multi-series already exists from task #785's earlier work, just call it. Don't build a parallel component.
- historicalActual + actualFromToday: solid line, existing balance color. Visually one continuous line through today.
- forecastFromToday: dashed line, distinct color (blue/grey suggestion). Reads as "projection."
- ReferenceLine at todayISO so the eye lands on the actual/forecast split.

TRIM TO REAL DATA, BOTH DIRECTIONS
- historicalActual ends at the last completed weekly bucket on or before today. No flat right-edge extension.
- forecastFromToday ends at whatever weekly bucket falls beyond the server's projection horizon. If the server projects 9 months out, the dashed line ends at 9 months — DO NOT carry the last projected value forward to fill 12. Document this choice in the code with a comment so the next person doesn't "fix" it.

LEGEND
- Two entries only: "Actual" (solid) and "Forecast" (dashed). Don't split historicalActual and actualFromToday in the legend — they're the same conceptual line.

TOOLTIP
- Pre-today bucket: actual value only.
- Today or future bucket: forecast value AND actual value (if a real balance point exists for that bucket) AND the delta between them.
- X-axis tick format reads cleanly across 12 months — monthly labels, not weekly.

CAPTION / SUBTITLE
- Caption: "Checking balance — actual vs forecast"
- Subtitle: "May 2026 – [month 12 months from today]"
